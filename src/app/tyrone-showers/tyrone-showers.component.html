Ty has been a technical subject matter expert for RESTful API design on my team since 2010. Ty authored the bulk of AT&T’s Guidelines for RESTful API Design. This document is an extremely useful compendium of best practices that we have used to drive consistency across APIs. The AT&T model has been to allow multiple diverse teams to create APIs that provide access to their technical services. In this model, it is critical that there be some standards that enable these teams to create APIs that can be easily consumed. This common approach allows developers consuming these APIs to understand and use these diverse services quickly and easily. In addition to authoring these standards for AT&T, Ty also sits as a member of the governing body that evaluated all APIs externally exposed by the company. Ty’s role on the “Northbound API Governance Board” has been to determine whether these emerging APIs adhere to RESTful guidelines and are appropriately useable for the developer community. Finally, in addition to authoring the standards and governing on their adherence, Ty plays an important consultative role with teams creating APIs. During the API design phase, countless teams have reached out for Ty’s guidance and he has helped them with a more effective RESTful design from the outset. This step has saved a tremendous amount of time by advising teams on how to pass the governance review ahead of time. Ty is a skilled API standards resource. However, I want to be sure to point out that Ty also has critical soft skills that make him a valuable member of the team. Ty has a very engaging personality and is a much more effective verbal communicator than most technical resources. I could see Ty doing an outstanding job as an API Evangelist or within Technical Sales. In summary, Ty has solid technical skills and outstanding people skills. I would strongly urge you to consider his candidacy for a wide range of positions.
